## 1. 遊戲總覽

- 2 人對戰。一場比賽由連續的「セット（Set）」組成（概念上 3~5 個 †1-2-2-1，實際由 Set 卡張數驅動）。
- **敗北條件**：「自己的セットカード為 0 張的狀態下，宣告ロスト」†0-1-3-1-1。也可隨時投降（投降不可被置換效果改寫 †0-1-3-2-1）。
- ラリー（Rally）＝從發球到某方宣告 Lost 之間的連續 turn †1-2-5。
- 每個 Set：發球權玩家先攻。Lost 的玩家從 Set 區拿 1 張卡進手牌；沒 Lost 的玩家獲得下個 Set 發球權。

## 2. 構築規則 †4-1-2

- 牌組**正好 40 張**；事件卡 **≤ 8 張**。
- **沒有同名卡張數限制**。禁止/制限卡表目前為空（floor rule 第15章）。
- 改變構築條件的卡片技能＝永續型技能，以置換效果方式適用（構築時就生效）†4-1-2-1。

## 3. 領域（Zones）†3

每位玩家各自擁有 10 個領域；「コート」＝サーブ/ブロック/レシーブ/トス/アタック 5 區的總稱 †3-1-2。

| 領域 | 公開性 | 備註 |
|---|---|---|
| デッキエリア | 非公開 | 不可看、不可改順序 |
| 手札 | 非公開 | 自己可看可排序；無上限 |
| セットエリア | 非公開 | 開局放 2 張背面朝上；被翻開的 Set 卡視為公開領域卡 †3-11-3；移出時可任選 †3-11-4 |
| ドロップエリア | 公開 | 棄牌堆；Master 可自由排序 |
| サーブ/レシーブ/トス/アタックエリア | 公開 | 疊放，最上面那張＝「キャラ」 |
| ブロックエリア | 公開 | 內分センターブロッカー置き場（疊放）與サイドブロッカー置き場 |
| イベントエリア | 公開 | 打出的事件卡放這裡（疊放） |

關鍵概念：
- **キャラ**＝コート各區「最上面」的角色卡 †1-2-14。
- **ガッツ（Guts）**＝コート各區キャラ「下面」的卡 †1-2-15。雙方可看內容；同區內順序 Master 可自由調整。Guts 卡的資訊原則上被無視（但「在卡下方時」觸發的技能仍有效）†1-2-15-2-1。
- 「Nガッツを払う」＝把指定區的 Guts N 張移到棄牌區；未指定時從該技能卡下方任選 †1-4-8。
- キャラ移動時，其下的 Guts 不跟著動；繼續效果不轉移給新露出的卡 †1-2-15-3。
- 卡疊上去時，原本的キャラ變成 Guts；繼續效果不被新卡繼承 †8-3。
- 移動到非コート領域＝洗掉原本套用的效果 †8-4-3；コート內區與區之間移動則效果保留 †3-1-5-1。
- 遊戲外（不在任何領域）的卡不被參照 †3-1-6。

## 4. 卡片資料 †2

- 卡片種類：キャラカード／イベントカード。
- 屬性（スキルで參照）：所属（可複數）、学年（可無/可複數）、ポジション（可複數）。
- 參數（パラメータ）：サーブ/ブロック/レシーブ/トス/アタックポイント 5 種。
  - **「－」不是 0**：不能比大小、不受加減影響 †1-3-2-1；對應區域參數為「－」的卡**不能登場到該區** †1-3-2-2。
  - 參數被加減後**可以是負數** †2-7-3。（對照：參數/OP/DP 以外的數值修正下限為 0 †0-2-9）
- イベントカード有プレイタイミングアイコン（指定可在哪些 Phase 的自由步驟打出）†2-12。
- **レアリティ無遊戲意義** †2-16-2。同卡番號不同卡面文字有差異時，基準レアリティ順序：N、R、S、頂、P †10-3-1。
- 官方勘誤/更新以官網為準 †2-19（→判例抓取要含勘誤）。

## 5. 回合結構（State Machine）†5

### 5.1 三種 Turn 構成 †5-2
1. **每個 Set 的第一個 turn**（發球權玩家）：サーブフェイズ → エンドフェイズ
2. **選擇攔網的 turn**：スタートフェイズ → ブロックフェイズ → エンドフェイズ
3. **選擇接球的 turn**：スタートフェイズ → ドローフェイズ → レシーブフェイズ → トスフェイズ → アタックフェイズ → エンドフェイズ

**攔網選擇限制（重要，総合ルール未明文）**：rules_sheet_v1 明寫「※相手のサーブやブロックでの返球に対しては、『ブロック』は選べません」——**只有對手 OP 來源是アタック時才能選攔網**；對手的發球、攔網回球只能選接球。OP 物件因此必須記錄來源種類（serve/block/attack），判例 Q498/Q500 也明確區分三種 OP。引擎實作：`canChooseBlock()`。相關設計佐證：關鍵字フェイント/ブロックアウト/ツーアタック的效果都是「妨礙下回合對手攔網」。

### 5.2 各 Phase 的內部步驟（CP＝チェックプロセス）

```
サーブフェイズ †5-5 : 開始[效果生效+待機+CP] → 登場St → 自由St → OP算出St → 終了[待機+CP+效果失效]
スタートフェイズ†5-6: 開始[…CP] → 選擇 Block 或 Receive（強制宣告）→ 終了[…]
ブロックフェイズ†5-7: 開始 → 登場St → 自由St → DP算出St → ジャッジSt → OP算出St → 終了[サイドブロッカー全部進棄牌區→待機+CP→效果失效]
ドローフェイズ †5-8 : 開始 → ドローSt(抽1) → 自由St → 終了
レシーブフェイズ†5-9: 開始 → 登場St → 自由St → DP算出St → ジャッジSt → 終了
トスフェイズ †5-10 : 開始 → 登場St → 自由St → 終了
アタックフェイズ†5-11: 開始 → 登場St → 自由St → OP算出St → 終了
エンドフェイズ †5-12: 開始[「ターン終了時」觸發，迴圈直到沒有新的待機] → クリンナップSt → 終了[CP→換 turn player→下個スタートフェイズ]
```

- 每個 Phase 開始：「○○フェイズ中」效果生效、「○○開始時」技能進入待機、跑 CP。
- 每個 Phase 結束：「○○終了時」技能待機、跑 CP、「○○フェイズ中/このフェイズ中」效果失效。
- エンドフェイズ特殊：同一 turn 已待機過的「ターン終了時」技能不重複待機；若還有未待機的，重複執行開始段 †5-12-2①。

### 5.3 各 Step 規格

- **登場ステップ †5-13**：turn player 從手牌登場角色卡。各 Phase 限制 †1-4-5-4-1：
  - サーブ：1 張到發球區
  - ブロック：1~3 張到攔網區；**同名卡不能登場 2 張以上**；其中 1 張必為センターブロッカー（若因效果不能放センター，最多只能登 2 張且都是サイド）
  - レシーブ：1 張到接球區
  - トス：1 張到托球區；**不可與レシーブキャラ同名**
  - アタック：1 張到攻擊區；**不可與トスキャラ同名**
  - 不登場（或無法登場）→ 自動宣告 Lost → ロストセットステップ。
- **自由ステップ †5-14**：迴圈：[CP] → turn player 選擇一項：使用主動型技能／打事件卡／宣告 Lost／pass。技能解決後回到 CP。
  - 只有直前的 Step 正常「終了」才會執行自由 St †5-14-2（被跳過就不執行）。
  - 能動宣告 Lost 的條件：非規則處理/效果解決中、且無待機中的被動技能/遲發效果 †1-4-9-2。
- **ジャッジステップ †5-15**：比較對手 OP vs 自己 DP → CP → OP/DP 消滅 → 失敗（DP < OP）則自動 Lost。效果可能加入額外的失敗條件；**任一失敗條件成立即失敗** †5-15-3。
- **OP 算出 †5-17**：サーブ＝發球角色サーブP；ブロック＝基準 0；アタック＝トスキャラのトスP＋アタックキャラのアタックP。算出後 **OP 與角色參數脫鉤**（之後參數變動不影響 OP）†5-17-7。OP 存活到「下個對手 turn 的ジャッジSt 結束」。
- **DP 算出 †5-18**：ブロック＝全攔網角色ブロックP合計；レシーブ＝接球角色レシーブP。同樣算出後脫鉤。
- **クリンナップステップ †5-19**：無指定期限的繼續效果結束；數值修正/固定效果結束。
- **ロストセットステップ †5-20**：①OP/DP 消滅 → ②「ロスト時」技能待機+CP → ③Set 中期限的繼續效果結束 → ④Set 中的待機狀態/遲發效果消滅 → ⑤進入インターバル。

### 5.4 インターバル †5-3
①開始（「インターバル中」效果生效）→ ②**雙方**補手牌到 6 張 → ③Lost 方從 Set 區拿 1 張進手牌 → ④Lost 方繼續效果結束 → ⑤非 Lost 方繼續效果結束 → ⑥非 Lost 方取得發球權，進入下個 Set 的サーブフェイズ。
（敗北判定：Set 區 0 張時宣告 Lost → 立即敗北，由規則處理 †8-2 在宣告當下攔截）

### 5.5 チェックプロセス †5-4（引擎核心迴圈）
```
loop:
  ① 把所有未處理的規則處理跑完（規則處理可在任何時點插入 †8-1-2）
  ② turn player 有待機中的被動技能/遲發效果 → 任選 1 個解決 → goto ①
  ③ 非 turn player 同上 → goto ①
  ④ 結束 CP，回到原本的進行
```

### 5.6 遊戲前準備 †4-2
提示牌組 → 隨機決定一方選擇「要/不要首發球權」→ 洗牌 → 各抽 6 張 → 從發球權方開始各可換牌一次（任意張放回洗勻補滿 6）→ 各從牌組頂放 2 張到 Set 區（背面）→ 發球權方サーブフェイズ開始。

## 6. 技能與效果系統 †1-3, †6

### 6.1 技能 4 型 †1-3-3
| 型 | 觸發 | 實作要點 |
|---|---|---|
| パーマネント型（永續） | 滿足發生條件就持續生效 | 無「宣言」；構築規則/開局變更類也屬此型 †6-5-4/5；條件不滿足立即失效 †6-3-1-1 |
| パッシブ型（被動） | 誘發條件 → 待機狀態 → CP 中解決 | 條件滿足 N 次就待機 N 次 †6-6-1①；移到非公開領域/被無效化/Phase 被跳過 → 不解決 †6-6-3, †8-4-4 |
| アクティブ型（主動） | 自由 St 中宣言使用 | 有使用時機 icon（指定 Phase）；解決後狀況完全不變則不可宣言 †6-7-1① |
| イベント型 | 打出事件卡時 | プレイ宣言 → 移到イベントエリア → 使用宣言+付 cost → 執行 †6-8 |

- **スキルコスト** †1-3-5：`〔…〕：` 表記。宣言時同時執行，付不出就不能宣言。技能被禁用時 cost 也不能付 †6-2-2。
- 卡片技能與總合規則矛盾時**技能優先**（除了：投降不可置換、登場禁止事項、領域外不參照）†0-2-2。
- 技能指示按卡面表記順序執行 †0-2-3。效果解決中不會插入其他技能的解決 †6-2-1。
- エリアアイコン †1-3-7：卡片在指定區域（サーブ/ブロック/レシーブ/トス/アタック/コート/手札）時技能才有效。

### 6.2 效果 3 種＋遲發性質 †1-3-4
- **即時効果**：解決中執行一次完畢。
- **継続効果**：有效期間內持續。預設期限＝クリンナップSt；明示期間則到期間結束 †0-2-11。
- **置換効果**：「A する場合、代わりに B する」。原事象不發生 †6-11-1-1（但只置換部分時其餘照常 †6-11-1-2）。同一事象多個置換效果：受影響的玩家決定順序，每個置換效果對一個事象最多適用 1 次 †6-11-2。選擇型置換（「してよい」）若選擇後無法執行則不可適用 †6-11-3。
- **遅発型効果**：以上任一種可附帶「之後滿足條件時再解決」性質，進待機狀態由 CP 處理。

### 6.3 數值修正系統（modifier layers）

適用順序 †6-10-1：
1. 卡面原本資訊（基準值）
2. 非數值變更的繼續效果（屬性/卡名變更等）全部適用
3. 數值變更的繼續效果全部適用

數值固定/修正優先順位 †0-2-12：
- 非永續效果：依解決順序套用，後解決的覆蓋先解決的
- 繼續效果 vs 永續技能的「固定」：**永續優先**
- 永續 vs 永續的「固定」：**先適用者優先**，其餘無視

期間終止條件 †0-2-11（任一成立即終止）：即時效果→クリンナップ；繼續效果→指示的期間（無指示→クリンナップ）；永續→條件不滿足時；對象是コート上キャラ→不再是キャラ時；對象是 OP/DP→該 OP/DP 消滅時。

數值域 †0-2-9：參數/OP/DP 可為負；其他數值下限 0。選數時除非另有指定，選 0 以上整數；「～まで」可選 0 †0-2-10。

### 6.4 重要解決原則
- **禁止優先** †0-2-6：指示與禁止同時存在 → 禁止贏。「○○のみ可能」＝「○○以外禁止」。
- **可能な限り実行** †0-2-5-5：部分不可執行就執行可能的部分；但「對象有指定而選不出合法對象」的部分視為不可執行（例：「不同卡名 2 張」只有 1 張時整段不執行）。
- 已是該狀態再被指示「變成該狀態」→ 不執行、不觸發「成為～時」†0-2-5-3。
- 同時行動：turn player 先 †0-2-7。規則處理同時發生→真同時；非規則的「同時」→逐一執行 †0-2-8。
- 解決時參照卡片**最終情報** †6-4：移入非公開領域/離開遊戲的卡 → 用最後可確認時的資訊。
- 領域移動誘發 †6-6-2：跨公開↔非公開時用「公開側」狀態的資訊；登場系被動技能只在「從別的領域移動而登場」時視為領域移動誘發。
- 已發生的效果指示獨立於發生源 †6-3-1（發生源離場/被無效化不影響已發生的指示；永續例外）。
- 永續技能：同一 turn 中曾脫離適用對象後再次符合 → 不再適用 †6-5-2。
- 全技能被無效的卡不能獲得新技能 †6-1-3。原生技能被無效＝存在但不發生；被付與的技能被無效＝消滅 †6-1-2。

### 6.5 定型表記（DSL 解析對照表）†7
- 無對象表記：對卡＝發生源自身；對玩家＝技能 Master †7-1-1。
- 「元々の」＝卡面未修正值。「【X】を持たない」「【X】以外の属性を持つ」「同じ/別々の」「まったく同じ」的嚴格定義見 †7-1-3~6。
- 「そうした場合」＝前句完整執行；「そうしなかった場合」＝前句有任何部分未執行 †7-2。
- 「以下から N 選ぶ」＋「▶」選項；同一選項不可選 2 次 †7-3-1。
- X：未定義時由 Master 任意決定；同一技能內所有 X 從付 cost 到解決完畢是同一個數 †7-3-3。
- 「次のターン/前のターン」：跨インターバル則該部分無視 †7-4-1。
- 公開/非公開領域查看規則 †7-5：效果結束時恢復非公開；看完未移動的要洗牌。
- 「スキルの影響を受けない」＝不被效果選為對象、執行時無視其存在；非直接對象時無關 †7-6-1。
- 「【行動】すれば使える」＝效果內分歧（不是 cost）；選擇使用則後續指示全部執行 †7-7-3。

## 7. 規則處理（State-Based Actions）†8
特定事象發生時自動插入執行、玩家不可控制：
- 敗北判定 †8-2
- 角色卡疊放 → 原キャラ變 Guts †8-3
- 領域變更處理 †8-4（非公開移動清待機等）
- 進行起因處理 †8-5（不登場→Lost、ジャッジ失敗→Lost、各算出步驟…）
- 跳過進行 †8-6：被跳過的 Phase/Step 內**什麼都不發生**（開始時/終了時不觸發、期限內到期的效果立即消滅、遲發效果消滅）；跳過ブロックフェイズ時サイドブロッカー仍要進棄牌區 †8-6-6。

## 8. 關鍵字 †9（v1.03 共 7 個）
- **ドシャットN** †9-2：攔網成功系遲發效果；該 turn 終了時觸發 →「自分の OP は N になる」（攔網軸 OP 基準 0 的替代）。
- **ワンタッチN** †9-3：即時效果：對手アタックOP −N，**並立即跳過進行中的ブロックフェイズ，移行到自己的ドローフェイズ**。
- **フェイントN** †9-4（v1.01 新增）：攻擊 turn 終了時觸發的遲發效果：自分 OP＝N，且下個對手 turn 不能登場攔網角色。
- **ブロックアウトN** †9-5（v1.01 新增）：攻擊後下個對手 turn，「元々のブロックポイント ≤ N 的攔網角色登場時」觸發 → 對手直接 Lost。
- **ターン1** †9-6（icon）：技能 A 解決了任何一部分 → 最後解決ターン1 → 該 turn 中「同卡名的自己的卡」技能全部無效化。
- **AパスN** †9-7（v1.03 新增）：該 turn 中托球角色登場時觸發 → 該托球角色トスポイント +N。

## 10. 引擎實作的推論註記（非官方條文）

- CP 是唯一的觸發解決點 → 實作成 `resolveCheckProcess(state)`，所有 Phase/Step 邊界呼叫。
- 「待機狀態」要記錄：來源卡、技能、觸發次數、所屬玩家 → CP 時依 turn player 優先逐一選擇解決（AI/玩家都要能選順序）。
- OP/DP 是獨立物件（非衍生值）：在算出 St 生成快照後，只受直接指定 OP/DP 的效果影響，存活期看 †5-17-6/†5-18-5。
- 修正層實作：`base → non-numeric continuous → numeric continuous(依解決順序) → permanent fix 覆蓋規則`。每個 modifier 記錄：來源、種類（add/set）、對象、期限。
- 隱藏資訊：牌組、手牌、Set 區（背面）。AI 模擬時需做 determinization。
- 「狀況完全不變則不可宣言技能」†6-7-1 對 legal move 產生器很重要（避免 AI 無限空使用技能）。
