## 0. 如何使用這份文件（執行者必讀）

1. **這份文件是介面的單一事實來源**。視覺、版面、互動、動畫的規格以此為準；遊戲規則與狀態結構以 `src/engine/types.ts` 為準。兩者衝突時停手開 issue，不要自行裁決。
2. **領域邊界（嚴格）**：你只能修改 `src/ui/**`、`src/index.css`、`src/App.tsx`、`public/**`（新增素材）。
   **禁止修改**：`src/engine/**`、`src/ai/**`、`data/**`、`tools/**`、`vite.config.ts`——這些由另一條 M3 工作線維護。UI 需要引擎提供新的查詢函式時，在 PR 描述中註明需求，由設計總監協調。
3. **驗收底線（每個 PR）**：`npx tsc --noEmit` 與 `npx vitest run` 全綠；既有對戰流程可完整跑完一場（發球→rally→得分→Set→勝負）；PR 附三個寬度的截圖（1280 / 834 / 390）。
4. **分期執行**：依第 9 節的 Phase 順序，一個 PR 對應一個子項。不要跨期合併實作。
5. 規格沒寫到的細節：選擇「最不增加視覺噪音」的做法，並在 PR 描述中列出你做的假設。

## 1. 設計願景

- **球場即介面**：對戰畫面的主體是一座俯瞰的排球場，上方對手、下方玩家。卡片區域按真實排球位置擺放——發球在端線外、接球在後場、舉球在三米線、進攻與攔網在網前隔網對峙。
- **卡片是主角**：介面整體暗沉（美式復古暗色調），讓彩色卡面成為畫面中最亮的東西。元件之間留足呼吸空間，不塞滿。
- **資訊就地呈現**：攻防數值（OP/DP）顯示在它發生的地方——球網上；階段高亮顯示在對應的場區上。玩家視線不需要離開球場。
- **點擊優先，拖放是糖**：所有操作都有 click 路徑（行動裝置相容的根基），拖放只作為桌面增強。

## 2. Design Tokens

全部定義為 CSS 變數，放在 `src/index.css` 的 `:root`。**任何元件不得出現硬編碼色票**——review 時 grep 十六進位色碼，tokens 之外出現即退回。

### 2.1 色彩

| Token | 值 | 用途 |
|---|---|---|
| `--bg-app` | `#26221f` | 應用最底層背景（暗暖棕黑） |
| `--bg-panel` | `#322d29` | 側欄面板、決策列、狀態列 |
| `--bg-panel-raised` | `#3d3631` | 面板上的浮起元素（hover 列、輸入框） |
| `--bg-out` | `#2b3442` | 球場外（界外區）暗藍灰 |
| `--court` | `#4b607f` | 球場地面（藍灰） |
| `--cream` | `#e8d8c9` | 球場線、主文字、我方按鈕底、卡片佔位 |
| `--ink` | `#26221f` | 米白底上的深色文字 |
| `--accent` | `#f3701e` | 強調色：OP/DP 徽章、選取外框、主要 CTA |
| `--p-me` | `#5f7fa6` | 我方識別色（log、徽章、Set 卡標記） |
| `--p-op` | `#c0584b` | 對方識別色（同上）＋危險動作按鈕 |
| `--text-dim` | `#a89c8e` | 次要文字 |
| `--text-faint` | `#6f655c` | 提示、停用文字 |
| `--gold` | `#d9a440` | 勝利、Guts 徽章 |

對比要求：`--cream` on `--bg-app` ≈ 9.8:1（AA 通過）；`--text-dim` on `--bg-panel` 需 ≥ 4.5:1，若字級 <14px 改用 `--cream`。

### 2.2 字體

| Token | 值 | 用途 |
|---|---|---|
| `--font-ui` | `"Noto Sans TC", -apple-system, sans-serif` | 全部中文與一般 UI |
| `--font-num` | `"IBM Plex Mono", "Courier New", monospace` | 計數器數字：OP/DP、牌組/棄牌張數、Set/Turn（復古計分板感；`font-variant-numeric: tabular-nums`） |

字重只用 400 / 500 / 700 三檔。標題（主選單 h1、勝負 banner）可加 `letter-spacing: 0.08em`。IBM Plex Mono 以 npm 套件 `@fontsource/ibm-plex-mono` 引入（不可用 CDN，離線可玩）。

### 2.3 形狀與質感

- 圓角：卡片 `6px`、面板 `8px`、徽章 pill `999px`。
- 邊框：一律 **1.5px 實線**（復古印刷的硬邊感），顏色用該區前景色的 35% 透明版。
- **禁止**：漸層、glow、blur、彩色陰影。卡片唯一允許的陰影是 `0 1px 0 rgba(0,0,0,.4)`（貼地感）。
- 紙紋 texture：暫不做（效能與複雜度不划算），列為未來可選。

### 2.4 間距

4px 基準刻度：`--sp-1: 4px`～`--sp-6: 24px`。場區之間最少 `--sp-3`(12px)，面板內距 `--sp-3`，欄與欄之間 `--sp-2`。

## 3. 版面結構（CSS Grid Areas）

對戰畫面根容器改為 named grid areas——**這是響應式的地基，Phase 0 的核心工作**：

```
桌面 ≥1024px                     直式 <768px
┌──────┬──────────┬──────┐      ┌────────────┐
│ left │  center  │ right│      │    hud     │  ← 單列 HUD
│ 240px│   1fr    │ 280px│      ├────────────┤
│      │          │      │      │   center   │  ← 球場（佔滿）
│      │          │      │      ├────────────┤
│      │          │      │      │  decision  │
│      │          │      │      ├────────────┤
└──────┴──────────┴──────┘      │    hand    │
                                └────────────┘
```

| 區域 | 內容 | <768px 的去向 |
|---|---|---|
| `left` | 階段指示、Set/Turn 計數、Game log、AI 速度控制 | 摺疊進 HUD（階段+OP）；log 變左上 icon 開抽屜 |
| `center` | 對方手牌列 → 球場 → 決策列 → 我方手牌 | 保留，球場等比縮放 |
| `right` | 卡片詳情、棄牌瀏覽面板 | 變右上 icon 開 bottom sheet；卡片詳情改長按觸發 |

斷點：`≥1024` 三欄；`768–1023`（iPad 直式 834 落此區）保留 center+right、left 摺疊為 HUD；`<768` 單欄。iPad 11" 橫式（1194）走三欄。球場用 `aspect-ratio` 維持比例，高度不足時整體縮小而不是裁切。

## 4. 球場規格（center 區核心）

座標以球場容器的百分比定位（容器 `position: relative`，場區 `position: absolute`），確保各斷點下相對關係不變。

| 場區 | 位置（我方半場；對方鏡像） | 備註 |
|---|---|---|
| 發球 | 端線外、水平置中（底部 out 區） | 卡可壓在端線上 |
| 接球 | 後場中央 | |
| 舉球 | 三米線上、中央偏右（左緣約 58%） | 雙方同側偏右，讓出中軸給 OP/DP |
| 進攻 | 網前左側（約 26%） | 正對對方攔網 |
| 攔網 | 網前右側（約 68%），**寬度需容納 3 張卡** | 中央攔網者置中、側翼兩張略小（參照現行 `blockSides` 渲染） |

球場繪製：界外區 `--bg-out` 滿版；場地 `--court` 內縮矩形、`--cream` 1.5px 實線描邊；中線（網）為貫穿容器全寬的 2px `--cream` 實線；兩條三米線為 1px 虛線。

### 4.1 四角卡區（每方各四堆）

| 角落 | 內容 | 互動 |
|---|---|---|
| 左角 | Set 卡區（卡背橫排，現行 `setArea`） | 我方在 pick-set-card 決策時可點選 |
| 右角 | 事件區・牌組・**棄牌堆**（三堆橫排） | 點棄牌堆 → 右欄開棄牌瀏覽（§7.1）；各堆下方以 `--font-num` 顯示張數 |

對方的四角鏡像在上半場。事件區張數上限 8，顯示最上一張＋張數。

### 4.2 球網 OP/DP 帶

網中線上的常駐徽章（`--accent` 底、`--ink` 字）：

- 無攻防時：隱藏（不要放「OP —」佔位，保持球場乾淨）。
- OP 產生時：`OP 14（發球）`＋**judge 預覽**：輪到我方防守選擇時，追加一行 `接球需 14+／攔網不可`（攔網可否由 `canChooseBlock()` 判定；需求值直接讀引擎判定邏輯，不要在 UI 重新實作規則）。
- DP 確定時：徽章短暫顯示 `14 vs 16` 比較後淡出（見 §8 動畫）。

### 4.3 對方手牌列

球場上方一條窄列：卡背縮圖（高約 32px）依張數排列＋張數數字。**這是對方所有出牌動畫的起點**。卡背素材見 §6。

## 5. 手牌與決策列

- 手牌卡寬基準 84px（桌面）；`<768px` 縮至 64px。超過容器寬時改為**重疊扇形**（負 margin 疊放，hover/tap 時該張上浮），不再用橫向捲動。
- 狀態：可用（正常）／不可登場（`opacity .35`＋去飽和）／選取中（`--accent` 2px 外框＋上移 6px）／多選序號徽章（攔網第 1 張標「中央」）。
- 決策列：手牌上方固定高度列（`--bg-panel`）。提示文字置左；按鈕置右——主要動作 `--cream` 底 `--ink` 字、危險動作（Lost／不登場）`--p-op` 底。**按鈕位置在各決策間保持一致**（肌肉記憶）。
- 鍵盤捷徑（Phase 4）：`Space`＝主要動作（Pass／確定）、`Esc`＝清除多選。**不**給 Lost 設快捷鍵（防誤觸）。

## 6. 素材規格（卡背）

| 素材 | 路徑 | 用途 |
|---|---|---|
| 通用卡背 | `decks/混合學校/cardback.png` | 預設卡背（復古格紋）。**Phase 0 將其複製為 `public/backs/default.png`** |
| 學校卡套 | `decks/<學校>/<學校>.png`（如 `decks/烏野/烏野.png`） | 該校牌組的專屬卡背，複製為 `public/backs/<學校>.png` |

規則：玩家牌組所屬學校有專屬卡套則用之，否則用通用卡背。`CardBack` 元件改為吃 `school?: string` prop 決定圖源。注意：部分學校尚無專屬卡套（之後由使用者重新設計），**程式必須優雅回退到通用卡背**（`onError` 換圖即可）。卡背圖較大（>500KB），以 `loading="lazy"`＋固定尺寸容器避免 CLS；不要求壓縮（本地工具）。

## 7. 盤面工具

### 7.1 棄牌瀏覽

- 入口：點任一方棄牌堆（§4.1）。
- 呈現：右欄切換為瀏覽面板（`<768px` 為 bottom sheet）：標題「你的棄牌（12）」、卡圖網格（每列 4 張）、**按進入棄牌的順序排列、最新在最上**。點卡 → 卡片詳情。`Esc` 或返回鍵關閉。
- 空狀態：「棄牌區是空的」置中、`--text-faint`。

### 7.2 Guts 堆疊檢視

場上卡堆右上角顯示 `G2` 徽章（`--gold` 底 `--ink` 字；G0 不顯示）。點徽章 → 浮動小面板展開整疊（由上而下＝由新到舊），各卡可 hover 看詳情。點面板外關閉。

### 7.3 回合歷史（log 升級）

log 每行保留現行著色（我方 `--p-me`、對方 `--p-op`）。新增：每個 rally 結束時插入一行分隔摘要（`Turn 4 ─ 你得分（攻擊 15 > 接球 12）`，`--font-num` 數字）。自動捲動維持；使用者手動上捲時暫停自動捲動，底部出現「回到最新」按鈕。

### 7.4 AI 速度控制

左欄底部四段切換：`0.5×／1×／2×／瞬間`，控制 AI 決策延遲（現行固定 650ms 改為 `650/speed`，瞬間＝0 且跳過動畫）。選擇存 `localStorage`。

## 8. 動畫規格

全部用 CSS transition/animation 或 FLIP 手法，**不引入動畫函式庫**。`prefers-reduced-motion: reduce` 時全部動畫改為直接到位。

| 動畫 | 觸發 | 行為 | 時長/easing |
|---|---|---|---|
| 卡片登場飛行 | 任一卡從手牌進入場區 | 從手牌位置飛至場區（FLIP：先改 state、再從舊位置 transform 過去）；對方的卡從對方手牌列起飛＋翻面（`rotateY` 90°換圖） | 280ms / `cubic-bezier(.2,.8,.3,1)` |
| 抽牌 | 牌組→手牌 | 卡背從牌組堆飛入手牌末端，落地翻面 | 220ms 同上；連抽間隔 60ms |
| 棄牌/Guts 沉降 | 卡進入棄牌堆或成為 Guts | 飛向目標堆，縮小至 0.5 淡出；Guts 則原地下沉 4px 變暗 | 240ms / ease-out |
| OP/DP 徽章 | 數值產生 | scale 0.6→1 彈出；數字滾動（`--font-num`） | 180ms / ease-out |
| 攻防判定 | DP 確定 | 徽章顯示 `14 vs 16`，勝方數字 `--gold` 閃一次，800ms 後淡出 | 淡出 300ms |
| 得分 banner | rally 結束 | 球場中央橫幅「BREAK!／得分」掃過（左進右出），同時 Set 計數跳動 | 900ms 全程 |
| 階段高亮 | phase 切換 | 場區邊框 `--cream`→`--accent` 過渡 | 150ms |

節奏原則：動畫不阻塞輸入——玩家可以在動畫播放中做下一個決策；AI「瞬間」模式跳過所有飛行動畫。

## 9. 實作分期（每期＝可獨立驗收的 PR 串）

### Phase 0 — 地基（不改變視覺以外的行為）
1. `index.css` 建立 §2 全部 tokens；既有色票全數替換為 tokens。
2. 對戰畫面改為 §3 grid areas；三個斷點骨架到位（內容未重排也要先能不破版）。
3. 卡背素材複製進 `public/backs/`；`CardBack` 改吃學校卡套＋回退。
驗收：三寬度截圖不破版、全綠、玩起來與改版前等價。

### Phase 1 — 換皮（wireframe 版面落地）
1. 球場繪製（§4）＋場區照位置表重排；四角卡區（含**棄牌堆與張數**）。
2. 左欄（階段/計數/log/速度控制佔位）、右欄（卡片詳情遷入）。
3. 對方手牌列；OP/DP 網中央徽章＋judge 預覽。
4. 決策列與手牌新樣式（§5，扇形重疊）。
驗收：與本文件 §3–§5 規格逐條核對；流程實測一整場。

### Phase 2 — 動畫與節奏
§8 全表＋ §7.4 AI 速度控制。驗收：reduced-motion 正常、瞬間模式不閃爍。

### Phase 3 — 瀏覽工具
§7.1 棄牌瀏覽、§7.2 Guts 檢視、§7.3 回合歷史。

### Phase 4 — 增強
拖放登場（桌面；click 路徑不得移除）、鍵盤捷徑、`<768px` 觸控細節（長按詳情、bottom sheet）、主選單與牌組編輯器換皮（沿用 tokens）。

## 10. 邊界情況

- **手牌 0 張**：手牌列顯示「沒有手牌」`--text-faint`，高度不塌。
- **手牌 15+ 張**：扇形重疊加密，最小可見寬 24px；hover/tap 上浮優先。
- **卡圖缺失**：沿用現行文字卡面（`card-text-face`），套新 tokens。
- **長卡名/技能文**：詳情面板不截斷（可捲動）；機翻標記（`badge-machine`）保留並改 token 配色。
- **Set 0 警告**：沿用現行邏輯（建局換牌階段不顯示），視覺改為 `--p-op` 文字。
- **gameOver**：勝負 banner 後決策列只剩「回主選單」；球場保持終局狀態可供檢視。

## 11. 無障礙

- 所有可點元素是 `<button>` 或帶 `role="button"`＋`tabIndex`；focus ring 用 `outline: 2px solid var(--accent)`。
- 場區帶 `aria-label`（如「我方攻擊區：日向翔陽，Guts 2」）；log 容器 `aria-live="polite"`。
- 純色之外都有文字/形狀冗餘（OP 來源用文字標注，不只靠顏色）。

## 12. 協作與版本管理（GitHub 工作流）

- **分支**：`ui/p<期>-<主題>`，例 `ui/p0-tokens`、`ui/p1-court`。基底 `main`。
- **PR 規範**：標題 `[UI Pn] 主題`；描述含：對應本文件章節、三寬度截圖、假設清單、全綠證明（tsc＋vitest 輸出）。
- **Review**：使用者＋設計總監（Claude）。Review checklist：①無硬編碼色票 ②未觸碰禁區檔案 ③click 路徑完整 ④斷點不破版 ⑤動畫尊重 reduced-motion。
- **衝突協調**：M3 工作線會動 `src/engine/**` 與 `data/cards.json`。UI 線只讀這些檔案，rebase 衝突時以 M3 為準。
- 規格疑義 → 開 GitHub issue 標 `ui-spec`，由設計總監裁決並回寫本文件（版本號遞增）。

## 13. 與 M3 效果系統的接縫（預留，暫不實作）

M3 已實裝技能（烏野全卡）。自由步驟將出現新的決策型別（技能使用、事件卡）。UI 預留：

- 決策列支援「動態選項清單」渲染（不只固定按鈕組）。
- 場上有可發動技能的卡 → 卡角顯示 `--accent` 小圓點；點卡 → 右欄詳情顯示技能與「發動」按鈕。
- 具體決策型別以屆時的 `engine/types.ts` 為準；此節由設計總監在 M3 介面穩定後補完規格。
